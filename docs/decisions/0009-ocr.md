# 0009. OCR via tesseract.js with invisible-text augmentation

- **Status:** Accepted
- **Date:** 2026-06-24

## Context

M7 adds OCR for scanned PDFs: recognize text, make it searchable/selectable,
keep it on save — all offline and without blocking the UI.

## Decision

- **Engine:** `tesseract.js` running in its own web worker (UI never blocks).
  All assets are bundled locally via `vite-plugin-static-copy` — the worker
  script, the wasm cores (`tesseract.js-core`), and the English data
  (`@tesseract.js-data/eng`, `eng.traineddata.gz`) — and served from the app
  origin, so OCR is **fully offline**. This fits the strict CSP: blob worker
  (`worker-src blob:`), wasm (`script-src 'wasm-unsafe-eval'`), and same-origin
  fetches (`connect-src 'self'`).
- **Pipeline:** render a page to an image with PDF.js → recognize → map each
  word's pixel box into PDF page space.
- **Augment, don't bolt on.** Rather than a parallel OCR text layer, OCR
  **embeds an invisible (opacity-0) text layer into the source PDF** at the word
  positions (pdf-lib) and reloads that source. The OCR text is then _real_ PDF
  text, so it flows through the existing text-layer selection, full-text search,
  and save paths with **no special-casing** — and it persists on save by
  construction.
- A `sourceRevision` counter forces the viewer to rebuild when a source's bytes
  are replaced.
- **Language:** English ships bundled; more languages are added by bundling the
  matching `@tesseract.js-data/<lang>` package (documented in ROADMAP).

## Consequences

- Scanned documents become searchable and selectable, and saving keeps the
  recognized text — verified end to end (render a text image → scanned PDF →
  OCR → assert selectable text).
- OCR augments the in-memory source; the original file on disk is unchanged
  until the user saves.
- Reusing the text-layer/search/save machinery kept the OCR surface small.

## Alternatives considered

- **A separate OCR overlay + bespoke search/highlight wiring** — more code and a
  second source of truth. Rejected in favor of augmenting the PDF text.
- **Fetching language data from a CDN on demand** — violates offline-first.
  Rejected; the data is bundled.
